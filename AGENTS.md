# Repository Expectations

## Context
- Read `docs/project_contract.md` before implementing planned sessions.
- Session-specific authority comes from `docs/session_{n}_contract.yaml`.

## Build And Test
- List exact commands here.
- State which commands are required after which kinds of changes.

## Boundaries
- Do not edit files outside the session contract blast radius without stopping.
- Do not add production dependencies without explicit approval.
- Do not change public API behavior unless the contract says so.

## Verification
- Classify failures before fixing.
- Prefer deterministic evidence over narrative claims.
- Final answers must mention checks run and checks not run.
